Digital Image Formation & Low-Level Processing

Quick Reference — Module 1

Geometric Transformations

Rotation, Scale & Shear Matrices
›
Rotation (θ): [cos θ -sin θ] [sin θ cos θ] Scale (sx, sy): [sx 0 ] [0 sy] Shear (hx, hy): [1 hx] [hy 1]

These are all components of affine transformations. Compose them via matrix multiplication.

Fourier Transform & Convolution
2D Discrete Fourier Transform (DFT)
›
F(u,v) = Σₓ Σᵧ f(x,y) · e^(-j2π(ux/M + vy/N)) Inverse: f(x,y) = (1/MN) Σᵤ Σᵥ F(u,v) · e^(j2π(ux/M + vy/N))

Key Properties:

Linearity Shift: f(x-x₀) ↔ e^(-j2πux₀/M)·F(u) Convolution Theorem Periodicity

Convolution Theorem: f*g in spatial domain = F·G in frequency domain (multiplication). This is why we do filtering in frequency domain — it's faster with FFT.

🎯 Low frequencies = smooth/DC component (center after shift). High frequencies = edges/details (periphery). Low-pass filter → blur. High-pass → sharpen/edge detect.
Convolution & Filtering
›
(f * h)(x,y) = Σₛ Σₜ f(s,t) · h(x-s, y-t)

Common Filter Kernels:

Image Enhancement & Restoration
Spatial Domain Enhancement
›

Enhancement is subjective (looks better to humans); Restoration is objective (reverse a known degradation).

TechniqueUse Case
Negative Transform: s = L-1-rEnhance white detail on dark images
Log Transform: s = c·log(1+r)Expand dark, compress bright (Fourier magnitude)
Power-Law (Gamma): s = c·rᵞMonitor gamma correction, general contrast
Median FilterRemove salt-and-pepper noise (non-linear)
Sharpening (Unsharp Mask)f + k·(f − blurred_f)
🎯 γ<1 → brightens. γ>1 → darkens. γ=1 → identity (linear). Median filter is best for impulse/salt-pepper noise; preserves edges better than mean.
Image Restoration — Wiener Filter
›

Degradation model: g = h * f + η (observed = blur * original + noise)

Wiener Filter (frequency domain): F̂(u,v) = [H*(u,v) / (|H(u,v)|² + Sη/Sf)] · G(u,v) Where: H = degradation function, Sη = noise PSD, Sf = signal PSD

When noise is zero → reduces to Inverse Filter. Wiener is optimal in MSE sense.

Histogram Processing
Histogram Equalization
›

Goal: make histogram uniform → maximize contrast.

sk = T(rk) = (L-1) · Σⱼ₌₀ᵏ p(rⱼ) Where: p(rⱼ) = nⱼ/n (probability of intensity rⱼ) L = number of gray levels (256) sk = output intensity
  1. Compute histogram: count pixels at each intensity
  2. Compute normalized histogram (probability)
  3. Compute cumulative distribution function (CDF)
  4. Map: new_intensity = round(CDF(k) × (L-1))
🎯 Histogram equalization is global. For local: apply equalization to sub-windows (CLAHE = Contrast Limited Adaptive HE). Expect a numerical problem in exam!
Histogram Specification (Matching)
›

Unlike equalization (uniform target), matching makes histogram look like a target histogram.

  1. Equalize input image → sk
  2. Equalize target/specified histogram → vk
  3. For each sk, find vk closest → create mapping
  4. Apply inverse of target's equalization transform

Binocular Stereopsis & Depth
Perspective Projection & Depth
›
Perspective Projection: x = f·X/Z, y = f·Y/Z (pinhole camera model) In homogeneous: [u v w]ᵀ = K · [R | t] · [X Y Z 1]ᵀ K = [f 0 cx] (intrinsic matrix) [0 f cy] [0 0 1]

Depth from Stereo:

Disparity: d = xL - xR (difference in x-coordinate between cameras) Depth: Z = f · B / d where B = baseline (distance between cameras), f = focal length
🎯 Larger disparity = closer object. If d=0, point is at infinity. Know the inverse relationship: Z ∝ 1/d.
Camera Calibration
›

Intrinsic Parameters (K): focal length (fx, fy), principal point (cx, cy), skew

Extrinsic Parameters [R|t]: rotation and translation — camera pose in world

Full projection matrix: P = K[R|t] (3×4 matrix)

🎯 Auto-calibration: find K from multiple images WITHOUT known calibration object (using epipolar constraints). Zhang's method uses planar checkerboard.
Epipolar Geometry
Epipolar Constraint & Fundamental Matrix
›

Key terms:

Epipole: projection of one camera center in the other image Epipolar plane: plane through X, C1, C2 Epipolar line: intersection of epipolar plane with image plane
Epipolar constraint: x'ᵀ F x = 0 Essential Matrix: E = [t]ₓ R (for calibrated cameras) Fundamental Matrix: F = K'⁻ᵀ E K⁻¹ (for uncalibrated, pixel coords) F is 3×3, rank 2, 7 DOF (scale invariant, det=0 constraint)

Why useful? Given point x in image 1, its correspondence in image 2 lies on the epipolar line l' = Fx — reduces 2D search to 1D.

🎯 E has 5 DOF (calibrated). F has 7 DOF (uncalibrated). To compute F: need ≥8 point correspondences (8-point algorithm).
Homography & Rectification
Homography
›

A Homography H is a 3×3 projective transformation mapping one image plane to another (8 DOF, scale invariant).

x' = H x (in homogeneous coordinates) Valid when: both cameras observe the same PLANE, OR cameras undergo pure rotation

Applications: panoramic stitching, rectification, planar AR, mosaicking.

🎯 Need minimum 4 point correspondences to compute H (each gives 2 equations, H has 8 unknowns). More points → use DLT with SVD.
DLT Algorithm (Direct Linear Transform)
›
  1. Normalize points (zero mean, unit scale) for numerical stability
  2. For each correspondence (xᵢ, x'ᵢ): form equations → build matrix A (2n × 9)
  3. Solve Ah = 0 using SVD: h = last column of V (singular vector for smallest σ)
  4. Reshape h (9×1) → H (3×3)
  5. Denormalize: H = T'⁻¹ · Ĥ · T
For each point: [x'ᵢ × H·xᵢ = 0] → 2 equations per point, 9 unknowns (entries of H) → Stacked into Ah=0 → SVD → h = null space of A
RANSAC & 3D Reconstruction
RANSAC Algorithm
›

Random Sample Consensus — robust estimation in presence of outliers.

  1. Randomly sample minimum number of points (e.g. 4 for H, 8 for F)
  2. Fit model (estimate H or F) from sample
  3. Count inliers: all points where |error| < threshold ε
  4. Repeat N iterations; keep model with most inliers
  5. Refit using all inliers of best model
Number of iterations: N = log(1-p) / log(1-(1-e)^s) p = desired probability (e.g. 0.99) e = outlier fraction s = sample size (min points to fit model)
🎯 RANSAC is robust to outliers (50%+ possible). More outliers → more iterations needed. Always paired with DLT or 8-point algorithm in exam questions.
3D Reconstruction Framework
›
  1. Detect & match features between images
  2. Estimate F (fundamental matrix) using RANSAC + 8-point
  3. Compute essential matrix E = Kᵀ F K
  4. Decompose E → [R|t] (4 solutions, pick correct by positive depth check)
  5. Triangulate 3D points using correspondences
  6. Bundle adjustment (non-linear refinement)

Edge Detection & Line/Corner Detectors
Canny Edge Detector
›

Optimal edge detector: good detection, good localization, single response.

  1. Smooth with Gaussian filter (remove noise)
  2. Compute gradient magnitude & direction (Sobel: Gx, Gy → |G|, θ)
  3. Non-Maximum Suppression (NMS): thin edges to 1-pixel width
  4. Double Thresholding: high threshold → strong edges; low → weak
  5. Hysteresis: weak edge kept only if connected to strong edge
🎯 NMS: at each pixel, compare gradient magnitude with neighbors in gradient direction. Keep only local maxima. This is what makes Canny edges thin and precise.
LoG & DoG (Laplacian / Difference of Gaussians)
›
LoG: ∇²G(x,y) = (x²+y²-2σ²)/σ⁴ · e^(-(x²+y²)/2σ²) DoG ≈ LoG: DoG = G(x,y,kσ) - G(x,y,σ) (used in SIFT!)

LoG finds edges as zero-crossings of second derivative. DoG is computationally cheaper approximation used extensively in SIFT for blob detection across scales.

Hough Transform (Lines & Circles)
›

Idea: Vote in parameter space. Each edge point votes for all lines/circles it could belong to.

Line (Normal form): ρ = x·cos θ + y·sin θ → Accumulator: A[ρ, θ] += 1 for each edge point → Peaks in A → detected lines Circle: (x-a)²+(y-b)² = r² → 3D accumulator space (a, b, r) → If r is known → 2D accumulator
  1. Detect edge points
  2. For each edge point, vote across all (ρ,θ) / (a,b,r)
  3. Find peaks in accumulator → detected shapes
🎯 Use normal (ρ,θ) form instead of (m,b) because m is unbounded for vertical lines. ρ ∈ [-diagonal, +diagonal], θ ∈ [0°, 180°].
Harris Corner Detector
›
Structure tensor M = Σ w(x,y) [Ix² IxIy] [IxIy Iy² ] Corner response: R = det(M) - k·trace(M)² = λ₁λ₂ - k(λ₁+λ₂)² k ≈ 0.04–0.06 (empirical constant)
λ₁, λ₂ valuesRegion type
Both smallFlat region (R ≈ 0)
One large, one smallEdge (R < 0)
Both largeCorner (R >> 0) ✓
🎯 Harris is rotation invariant (uses eigenvalues) but NOT scale invariant. For scale invariance → use Hessian-Affine or SIFT.
Feature Descriptors: SIFT, SURF, HOG, GLOH
SIFT — Scale Invariant Feature Transform
›

Invariant to: scale, rotation, illumination, viewpoint (partially).

  1. Scale-space extrema: Build DoG pyramid, find local max/min across scale and space
  2. Keypoint localization: Fit quadratic to refine position, remove low contrast/edge points
  3. Orientation assignment: Histogram of gradients in neighborhood → dominant orientation (makes rotation invariant)
  4. Descriptor: 4×4 spatial cells × 8 orientation bins = 128-dim vector
Descriptor: 128-dimensional, L2-normalized Matching: Lowe's ratio test: d1/d2 < 0.8 → good match
🎯 SIFT is patented (historically). OpenSURF/ORB are free alternatives. In exam: remember 128 dimensions, DoG for detection, gradient histogram for descriptor.

HOG (Histogram of Oriented Gradients) is used primarily for object detection (pedestrian detection with SVM). Works on dense grids, not keypoints.

HOG pipeline: normalize image → compute gradients → cell histograms (8 bins) → block normalization → concatenate → feature vector
Scale-Space, Image Pyramids & Gabor Filters
Scale-Space & Image Pyramids
›
Scale-space: L(x,y,σ) = G(x,y,σ) * I(x,y) where G is Gaussian with std σ Gaussian Pyramid: repeatedly Gaussian blur + downsample (by 2) Laplacian Pyramid: difference between adjacent Gaussian levels Lᵢ = Gᵢ - upsample(Gᵢ₊₁)

Laplacian pyramid captures band-pass details at each scale. Used for image compression, blending, and feature detection across scales.

🎯 SIFT uses DoG pyramid (approximation of LoG pyramid). Gaussian pyramid discards information; Laplacian pyramid is lossless (you can reconstruct original).
Gabor Filters & DWT
›
Gabor Filter: g(x,y,λ,θ,σ) = Gaussian × sinusoidal grating = exp(-(x'²+γ²y'²)/2σ²) · cos(2πx'/λ + ψ) Parameters: λ=wavelength, θ=orientation, σ=scale, γ=aspect ratio

Gabor filters are optimal in joint spatial-frequency localization (like human visual cortex V1 cells). Used for texture analysis.

DWT (Discrete Wavelet Transform): Decomposes image into approximation (LL) + detail (LH, HL, HH) subbands at multiple scales. Better than Fourier for non-stationary signals.

Image Segmentation Methods
Region Growing & Edge-Based Segmentation
›

Region Growing: Start from seed pixel, add neighbors that satisfy similarity criterion (intensity, color, texture). Stop when no more pixels qualify.

Edge-Based: Detect edges (Canny etc.) → try to form closed contours → fill contours as regions. Problem: edges are rarely perfectly closed.

🎯 Region growing is sensitive to seed selection. Edge-based fails with broken edges. Both are classic methods, often combined with modern approaches.
Graph-Cut Segmentation
›

Model image as a graph: pixels=nodes, edges=similarity weights. Segmentation = finding minimum cut separating foreground (source S) from background (sink T).

Energy = Σ Data_term(pixel assignment) + λ·Σ Smoothness_term(neighbors) Minimized via max-flow / min-cut (equivalent by max-flow min-cut theorem)

GrabCut = iterative graph cut with GMM color models. Used in Adobe Photoshop.

🎯 Graph-Cut is a global optimization method. Unlike region growing (greedy), it finds the globally optimal binary labeling given the energy function.
Mean-Shift Segmentation
›

Non-parametric: finds modes (peaks) of the density function in feature space (color + spatial).

  1. Represent each pixel as point in feature space (e.g., [R,G,B,x,y])
  2. For each point: compute mean of points within bandwidth h
  3. Shift point to that mean; repeat until convergence
  4. Points converging to same mode → same segment

No need to specify number of clusters (unlike k-means). Bandwidth h controls granularity.

MRF & Texture Segmentation
›

Markov Random Field (MRF): Statistical model where each pixel's label depends only on its neighbors (Markov property). Energy minimization via belief propagation or graph cut.

P(label | image) ∝ exp(-E(label, image)) E = Σ_i Ei(li) + Σ_{i,j∈N} Eij(li, lj) (unary: data fit) + (pairwise: smoothness)

Texture Segmentation: Compute texture features (Gabor filters, LBP, GLCM) per region, then cluster/classify. Key idea: similar texture → same segment.
