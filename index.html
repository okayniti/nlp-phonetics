What is a Speech Sound?

In linguistics and NLP, speech sounds are the smallest units of spoken language. Every word you say is made up of a sequence of these sounds. The study of speech sounds is called phonetics.

There are three branches of phonetics that NLP draws from:

Consonants vs Vowels

Every sound is classified along two axes:

Type Definition Example Words
Vowels Air flows freely through the vocal tract; no obstruction a, e, i, o, u
Consonants Airflow is partially or fully obstructed b, p, t, d, s, z …

Consonants are further classified by place of articulation (where the blockage occurs), manner (how it's blocked), and voicing (vocal cords vibrate or not).

Voiced vs Voiceless — Minimal Pairs

A minimal pair is two words differing in exactly ONE sound. This is key to identifying phonemes.

The Phoneme

A phoneme is the smallest unit of sound that can change meaning in a language. It's an abstract, mental category — not a physical sound.

Example: In English, /p/ is a phoneme. "pat" ≠ "bat" because /p/ ≠ /b/. Changing the phoneme changes the meaning.

English has approximately 44 phonemes but only 26 letters — hence we need IPA!

Phoneme vs Allophone

A phoneme can be realized in different ways depending on context — these variants are called allophones. They don't change meaning; they're just different pronunciations of the same phoneme.

What are Phonological Rules?

Phonological rules describe how phonemes change in certain environments. They explain why we don't pronounce words in isolation the same way we do in natural speech.

Rules are written in the form:

A → B / C __ D
sound A becomes B when it occurs after C and before D

These rules are critical for NLP systems like TTS (text-to-speech), ASR (speech recognition), and G2P (grapheme-to-phoneme) converters.

Key Phonological Rules
01
Assimilation
A sound becomes similar to an adjacent sound. E.g., "in" + "possible" → "impossible" (/n/ → /m/ before bilabial /p/)
02
Deletion (Elision)
A sound is dropped in certain contexts. E.g., "every" → "ev'ry" in fast speech. The /ə/ is deleted between consonants.
03
Insertion (Epenthesis)
A sound is added. E.g., "athlete" → "ath-uh-lete" in some dialects. Helps with difficult consonant clusters.
04
Flapping (American English)
/t/ or /d/ → [ɾ] (flap) between a stressed and unstressed vowel. "butter", "water", "ladder" all have the same middle sound!
05
Vowel Reduction
Unstressed vowels are reduced to schwa [ə]. E.g., "banana" → /bəˈnænə/. Very common in English; critical for TTS models.
06
Final Devoicing
Voiced consonants become voiceless at the end of a word. Common in German/Russian. E.g., German "Rad" (wheel) is pronounced like "rat".
Formal Rule Notation

In computational phonology, rules use feature bundles and context markers. Here are examples:

/t, d/ → [ɾ] / Vstressed __ Vunstressed
Flapping: t/d becomes a flap between stressed and unstressed vowels
/n/ → [m] / __ [+bilabial]
Assimilation: n becomes m before bilabial consonants
V → [ə] / __ (in unstressed syllable)
Vowel reduction: any vowel reduces to schwa when unstressed
# Simple Python G2P rule application import re def apply_flapping(phonemes): # t/d → flap between vowels (simplified) pattern = r'([AEIOU])([td])([aeiou])' result = re.sub(pattern, r'\1ɾ\3', phonemes) return result # "butter" → /bʌtər/ → /bʌɾər/ print(apply_flapping("bʌtər"))
Why Phonetics Matters in NLP

Every major speech NLP system relies on phonetic knowledge. Here's where each concept shows up:

NLP Task Phonetic Concept Used
Speech Recognition (ASR) Phoneme models (HMMs, CTC), acoustic features (MFCCs)
Text-to-Speech (TTS) G2P conversion, phonological rules, prosody
Spell Checking Phonetic similarity (Soundex, Metaphone algorithms)
Named Entity Recognition Phoneme patterns help identify proper nouns
Machine Translation Loanword pronunciation; cross-lingual phoneme mapping
Dialogue Systems Homophone disambiguation, prosody for intent detection

Soundex — Phonetic Hashing for NLP

Soundex is a classic phonetic algorithm that maps names to codes based on sound, ignoring spelling. Used in spell checkers, fuzzy search, and duplicate detection.

# Soundex groups phonetically similar consonants # B,F,P,V → 1 | C,G,J,K,Q,S,X,Z → 2 | D,T → 3 # L → 4 | M,N → 5 | R → 6 # "Smith" and "Smythe" → same Soundex code S530 # "Johnson" and "Jonson" → J525 import jellyfish print(jellyfish.soundex('Smith')) # → S530 print(jellyfish.soundex('Smythe')) # → S530 ✓ Same!
MFCCs — Feeding Phonemes into ML Models

Mel-Frequency Cepstral Coefficients (MFCCs) are the most common audio features used in ASR and emotion detection — including your voice emotion project!

They approximate how humans perceive pitch and represent acoustic phoneme information as a vector fed into neural networks (ANN, CNN, LSTM).

import librosa import numpy as np audio, sr = librosa.load('speech.wav', sr=16000) mfcc = librosa.feature.mfcc(y=audio, sr=sr, n_mfcc=13) # shape: (13, time_frames) # Each column = one "snapshot" of phoneme-like features # Feed into LSTM for sequence modeling of speech
